Basic Git Operations

>> git init

This command initializes a .git file inside the directory where this command is called.

>> git add [fileName]

This command adds the file, to which some changes have been made to the staging area.

Note: If changes are made to multiple files, to add all the changes to the staging area, replace the "[fileName]" with "." .

command :- >> git add .

>> git status

Git status gives us an insight of the status of the different project files.

>> git log

Git log prints the previous commits done on the project from which the project history can be identified.

>> git diff

git diff is the command which helps us visualize the difference between the saved file versions

>> git diff -r HEAD [fileName]

To find the find all the differences in a specific file in comparison to it's last saved version, this command is used.
To find the differences in all the files in comparison to the last saved version, run the same command omitting the [fileName]

>> git branch [-a, optional flag]

The 'git branch' command displays the local branches only. The 'git branch -a' command displays all the local as well as the renote branches.

git checkout [branchName]

This command is used to switch between branches.

>> git branch -d [branchName]

To delete a local branch
For example, to delete a local branch name "fix", we will use the following command. >> git branch -d fix

>> git push [origin] --delete [branchName]

To delete a remote branch
