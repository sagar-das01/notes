Building A Cluster

Steps

  1. Building a cluster requires installing all the Runtime and Kubernetes configuration packages.
  2. Then create the cluster.
  3. Configure Pod Networking.
  4. Join Nodes to your cluster.

Required Packages

Following are the required packages to build a cluster

  • containerd or docker
  • kubelet
  • kubeadm
  • kubectl

These are the 4 components which must be installed in all the nodes regardless of the control plane node or the cluster node.

Containerd/Docker

This is the container runtime environment

Kubelets

Kubelet is the component which drive the individual nodes in the cluster

Kubeadm[Kubeadmin]

Kubeadm bootstraps the cluster, i.e. it makes the cluster components up and running.

This is also responsible for adding new nodes to the cluster.

kubectl[Kube Control]

This is the primary command line tool to adminsiter all the workload in the cluster.

Steps to get and install Kubernetes on Ubuntu

Following are the steps to install Kubernetes on any Virtual Machine, Windows or Linux

VM Installation

  1. install containerd
    2. sudo apt-get install -y containerd\
  2. add the gpg key
    3. curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
  3. adding kubernetes to the source list
    4. cat </etc/apt/sources.list.d/kubernetes.list deb https://apt.kubernetes.io/ kubernetes-xenial main
  4. update linux to add the changes and install any necessary dependencies
    5. sudo apt get update
  5. install kubelet, kubeadm and kubectl
    6. apt-get install -y kubelet kubeadm kubectl
  6. Configure the packages to prevent auto upgrade
    7. apt-mark hold kubelet kubeadm kubectl containerd

    apt-mark sets the settings for the package

    holdholds back the package(s) and saves it from any auto upgrade
